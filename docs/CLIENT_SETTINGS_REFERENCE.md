# Справочник настроек клиента (для подсказок в UI)

Подробное описание **всех** параметров, которые клиент отправляет на сервер.
Источник истины — `crates/screener/src/config.rs` (`ClientConfig`) и сообщение
`watch` (`crates/api/src/session.rs`). Технический протокол — в
[CLIENT_INTEGRATION.md](CLIENT_INTEGRATION.md).

У каждого параметра есть:
- **🏷 Подсказка** — короткая строка для tooltip при наведении;
- **Описание** — развёрнутый текст для кнопки «подробнее»;
- **Тип / По умолчанию / Единица / Диапазон** — для валидации на клиенте.

## Общие правила формата (важно для UI)

- **Все денежные и процентные значения — строки JSON** (десятичная точность), например `"0.02"`, а не число. На клиенте парсить desimal-типом, не float.
- **Проценты — это доли, а не проценты:** `"0.02"` = **2%**. В UI показывайте пользователю проценты, но на сервер шлите долю.
- **Время — в миллисекундах** (`ms`).
- **Биржи — строки в нижнем регистре:** `bybit, okx, mexc, bitget, gate, coinex, kucoin, phemex`.
- Все поля **необязательные** — что не прислали, берётся из серверных дефолтов.

## 1. Выбор рынков

### `exchanges`
- **Тип:** массив строк · **По умолчанию:** все 8 · **Значения:** `bybit, okx, mexc, bitget, gate, coinex, kucoin, phemex`
- **🏷 Подсказка:** Какие биржи участвуют в поиске спреда.
- **Описание:** Список бирж, между которыми ищем межбиржевой спред. Спред считается только между разными биржами из этого списка. Меньше бирж — меньше пар и меньше шума, но и меньше возможностей. Для реального арбитража имеет смысл держать только те биржи, где у вас есть счёт.

### `quote`
- **Тип:** строка · **По умолчанию:** `"USDT"`
- **🏷 Подсказка:** Валюта расчёта (обычно USDT).
- **Описание:** Котируемая/расчётная валюта перпа (маржа). В Фазе 1 поддерживается USDT. Также используется фильтрами перевода (`require_transferable`) — проверяется возможность перевести именно эту валюту между биржами.

### `allow_symbols`
- **Тип:** массив строк · **По умолчанию:** `[]` (все)
- **🏷 Подсказка:** Белый список монет (пусто = все).
- **Описание:** Если непусто — скринятся только эти базовые активы (например `["BTC","ETH","ARB"]`). Пусто — берутся все монеты из авто-обнаруженной вселенной (листятся ≥2 бирж). Регистр не важен.

### `deny_symbols`
- **Тип:** массив строк · **По умолчанию:** `[]`
- **🏷 Подсказка:** Чёрный список монет.
- **Описание:** Базовые активы, которые исключаются из скрининга, даже если проходят по остальным фильтрам. Имеет приоритет над `allow_symbols`.

## 2. Ликвидность

### `min_24h_quote_volume`
- **Тип:** decimal-строка · **По умолчанию:** `"0"` (выкл.) · **Единица:** USDT
- **🏷 Подсказка:** Отсекать неликвид по суточному объёму.
- **Описание:** Минимальный суточный объём торгов (в USDT). Пара отсеивается, если объём на обеих ногах ниже порога. Отсекает «мёртвые» рынки — главный источник фейковых спредов. Проверяется по данным тикера там, где они есть (сейчас надёжно — Bybit); если объём неизвестен на обеих биржах, фильтр пропускается. `0` — фильтр выключен.

### `min_open_interest`
- **Тип:** decimal-строка или `null` · **По умолчанию:** `null` (выкл.) · **Единица:** базовые единицы
- **🏷 Подсказка:** Минимальный открытый интерес по перпу.
- **Описание:** Порог открытого интереса (OI) — специфичный для перпов индикатор ликвидности. Низкий OI = тонкий рынок, спред трудно исполнить. Как и объём, применяется там, где биржа отдаёт данные, иначе пропускается.

## 3. Полоса спреда (главный контроль 2–20%)

### `min_net_spread_pct`
- **Тип:** decimal-строка (доля) · **По умолчанию:** `"0.02"` (2%) · **Диапазон:** `≥ 0`
- **🏷 Подсказка:** Нижний порог чистого спреда для сигнала.
- **Описание:** Минимальный **чистый** спред (после комиссий обеих ног), при котором приходит сигнал. Ниже — не сигналим. Типичные значения 0.005–0.05 (0.5–5%).

### `max_net_spread_pct`
- **Тип:** decimal-строка (доля) · **По умолчанию:** `"0.20"` (20%) · **Диапазон:** `≥ min_net_spread_pct`
- **🏷 Подсказка:** Верхняя отсечка «призрачных» спредов.
- **Описание:** Спреды выше этого порога почти всегда «призраки» — делистинг, заморозка вывода, разные токены под одним тикером. Такие сигналы **не отправляются** (это не возможность, а ошибка данных). Ключевой фильтр «мираж vs реальность».

### `target_notional_q`
- **Тип:** decimal-строка · **По умолчанию:** `"2000"` · **Единица:** USDT · **Диапазон:** `> 0`
- **🏷 Подсказка:** На какой размер считаем исполнимость.
- **Описание:** Размер сделки (в USDT), на который считается **исполнимый** спред: сервер идёт по стакану и берёт средневзвешенную цену (VWAP), реально достижимую на этот объём, а не «лучшую цену» с копеечным объёмом. Это и превращает красивую цифру в реально захватываемую. Больше размер — честнее оценка, но меньше монет пройдут по глубине.

### `min_executable_notional`
- **Тип:** decimal-строка · **По умолчанию:** `"500"` · **Единица:** USDT
- **🏷 Подсказка:** Минимум реальной глубины на обеих ногах.
- **Описание:** Требуемая минимальная исполнимая глубина на обеих ногах. Если стакан не может набрать хотя бы столько — сигнала нет. Защищает от «бумажных» спредов на 50 долларов глубины.

### `depth_levels_n`
- **Тип:** целое · **По умолчанию:** `20` · **Диапазон:** `> 0`
- **🏷 Подсказка:** Сколько уровней стакана учитывать.
- **Описание:** Сколько уровней стакана прошивать при расчёте VWAP. Реальная глубина ограничена каналом биржи (у части бирж это 5 уровней, у Bybit — до 50), поэтому очень большие значения обрезаются автоматически.

## 4. Комиссии и фандинг

### `taker_fee`
- **Тип:** объект `{ "биржа": "доля" }` · **По умолчанию:** ставки из конфига сервера, запасная `"0.0006"` (0.06%)
- **🏷 Подсказка:** Комиссия тейкера по каждой бирже.
- **Описание:** Комиссия тейкера по каждой бирже (доля: `"0.0005"` = 0.05%). Вычитается с **обеих** ног при расчёте чистого спреда. Пример: `{ "bybit": "0.00055", "okx": "0.0005" }`. Для не указанных бирж берётся серверная ставка. Серверные дефолты: bybit 0.00055, okx 0.0005, mexc 0.0002, bitget 0.0006, gate 0.0005, coinex 0.0005, kucoin 0.0006, phemex 0.0006.

### `include_funding_diff`
- **Тип:** булево · **По умолчанию:** `true`
- **🏷 Подсказка:** Считать сигнал по разнице фандинга.
- **Описание:** Включает отдельный сигнал по **разнице ставок фандинга** между биржами (лонг там, где фандинг ниже; шорт — где выше). Латентность-толерантная стратегия: держим дельта-нейтральную пару и собираем разницу.

### `min_funding_diff_apr`
- **Тип:** decimal-строка (доля, годовых) · **По умолчанию:** `"0.15"` (15% годовых)
- **🏷 Подсказка:** Минимальная годовая разница фандинга.
- **Описание:** Минимальная **аннуализированная** разница фандинга, при которой фандинг-сигнал попадает в событие. Ниже — не показываем как фандинг-возможность.

### `funding_hold_hours`
- **Тип:** decimal-строка · **По умолчанию:** `"8"` · **Единица:** часы
- **🏷 Подсказка:** На сколько часов держим позицию (для расчёта фандинга).
- **Описание:** Предполагаемое время удержания позиции для оценки фандинг-выгоды. Влияет на пересчёт ожидаемого дохода от фандинга.

## 5. Фильтры реальности (защита от миражей)

### `require_transferable`
- **Тип:** булево · **По умолчанию:** `false`
- **🏷 Подсказка:** Требовать ввод/вывод валюты расчёта на обеих биржах.
- **Описание:** Требует, чтобы валюта расчёта (USDT) имела **включённые ввод и вывод** на обеих биржах — иначе маржу нельзя перекинуть между площадками, и «арбитраж» неисполним. По умолчанию выключено: в Фазе 1 публичные данные о переводах есть не для всех бирж (Bybit/OKX/MEXC требуют ключей). Включайте, только если для ваших бирж наполнен transfer-store.

### `require_common_network`
- **Тип:** булево · **По умолчанию:** `false`
- **🏷 Подсказка:** Требовать общую сеть перевода.
- **Описание:** Требует, чтобы у обеих бирж была хотя бы одна общая включённая сеть (chain) для валюты расчёта — иначе перевод между ними невозможен физически. Те же оговорки Фазы 1, что и выше.

### `max_book_age_ms`
- **Тип:** целое · **По умолчанию:** `3000` · **Единица:** ms
- **🏷 Подсказка:** Максимальный «возраст» котировки.
- **Описание:** Котировки старше этого возраста (по локальному времени приёма, не по часам биржи) исключаются из расчёта — «залипшая» сторона не даст ложный спред. Меньше значение — свежее данные, но выше шанс временно потерять биржу.

## 6. Динамика спреда (сигнатура «тугая база + всплески»)

Хороший арбитражный спред бо́льшую часть времени тугой (0–1%) и **изредка** всплёскивает, а не висит широким постоянно. Эти фильтры отделяют реальные всплески от структурных ловушек.

### `enable_dynamics`
- **Тип:** булево · **По умолчанию:** `true`
- **🏷 Подсказка:** Включить фильтры по динамике спреда.
- **Описание:** Главный выключатель фильтров ниже. Выключите, чтобы получать сигналы только по мгновенному спреду, без учёта его истории.

### `max_baseline_spread_pct`
- **Тип:** decimal-строка (доля) · **По умолчанию:** `"0.01"` (1%)
- **🏷 Подсказка:** Отсекать «вечно широкие» спреды.
- **Описание:** Если **базовая линия** (медиана спреда за окно) выше этого порога — монета отсеивается: постоянно широкий спред это, как правило, структурная поломка (разный токен / мёртвая сторона), а не возможность, потому что он не сходится.

### `min_spike_z`
- **Тип:** decimal-строка · **По умолчанию:** `"3"` · **Единица:** стандартные отклонения (σ)
- **🏷 Подсказка:** Насколько сильным должен быть всплеск.
- **Описание:** Текущий спред должен быть минимум на столько σ выше своего среднего — это отличает настоящий выброс от «оно всегда широкое». Выше значение — строже отбор, реже и качественнее сигналы.

### `max_spread_duration_ms`
- **Тип:** целое · **По умолчанию:** `300000` (5 мин) · **Единица:** ms
- **🏷 Подсказка:** Отсекать спреды, висящие широкими слишком долго.
- **Описание:** Если спред держится выше порога дольше этого времени — сигнал отсекается: здоровый арбитраж закрывается быстро, а долго висящий широкий спред обычно ловушка (капитал застрянет).

### `min_dynamics_samples`
- **Тип:** целое · **По умолчанию:** `20`
- **🏷 Подсказка:** Прогрев статистики до применения фильтров.
- **Описание:** Пока по монете накоплено меньше стольких отсчётов истории — фильтры динамики не применяются (чтобы не резать сигналы на «холодном старте»).

## 7. График (поток watch)

### `max_chart_spread_pct`
- **Тип:** decimal-строка (доля) · **По умолчанию:** `"0.50"` (50%)
- **🏷 Подсказка:** Порог отсева аномалий на графике.
- **Описание:** Персональный порог отсева аномалий для **графика спреда**: точки backfill и live-тики, у которых `abs(net_pct)` выше порога, не отправляются этому клиенту (wrong-token / битая котировка — ошибка данных, а не сигнал). Сервер также держит глобальный жёсткий кап; этот параметр только ужесточает его. Чтобы поменять на активном графике — пришлите новый `subscribe` и повторный `watch` (порог фиксируется в момент `watch`).

## 8. Контроль шума (антиспам)

### `hysteresis_step_pct`
- **Тип:** decimal-строка (доля) · **По умолчанию:** `"0.005"` (0.5%)
- **🏷 Подсказка:** Насколько должен подрасти спред для повторного сигнала.
- **Описание:** После первого сигнала повторный придёт только если спред раздвинулся **ещё** минимум на этот шаг относительно предыдущего пика. Защита от спама повторами при мелких колебаниях.

### `min_signal_lifetime_ms`
- **Тип:** целое · **По умолчанию:** `1500` · **Единица:** ms
- **🏷 Подсказка:** Сколько спред должен продержаться до сигнала.
- **Описание:** Спред должен непрерывно держаться выше порога хотя бы столько времени, прежде чем придёт сигнал — отсекает мгновенные мерцания, которые невозможно исполнить.

### `cooldown_ms`
- **Тип:** целое · **По умолчанию:** `2000` · **Единица:** ms
- **🏷 Подсказка:** Пауза между сигналами по одной монете.
- **Описание:** Минимальная пауза между двумя сигналами по одному и тому же инструменту.

### `max_signals_per_min`
- **Тип:** целое или `null` · **По умолчанию:** `120` · **Единица:** сигналов/мин
- **🏷 Подсказка:** Лимит сигналов в минуту.
- **Описание:** Общий лимит числа сигналов в минуту на сессию. `null` — без лимита. Защищает клиента от потока при рыночной турбулентности.

## 9. Параметры графика (в сообщении `watch`, не в `ClientConfig`)

Эти два поля приходят в сообщении `watch` при открытии графика монеты (см. [CLIENT_INTEGRATION.md](CLIENT_INTEGRATION.md) §2.45).

### `window_ms`
- **Тип:** целое или `null` · **По умолчанию:** `900000` (15 мин) · **Единица:** ms
- **🏷 Подсказка:** Окно графика (сколько истории показывать).
- **Описание:** Сколько истории подгрузить в график сразу (backfill) и держать в окне. Ограничивается серверным максимумом (по умолчанию 30 мин).

### `resolution_ms`
- **Тип:** целое или `null` · **По умолчанию:** сервер выбирает (обычно 1000)
- **🏷 Подсказка:** Желаемая частота точек (сервер может ограничить).
- **Описание:** Желаемый интервал между точками. Сервер семплит на фиксированной частоте и **возвращает фактическое значение** в `watch_snapshot.resolution_ms` — на него и ориентируйтесь.
